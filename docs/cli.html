Quick Start

bazinga "What is consciousness?"

That's it. Works immediately. No API keys needed.

--ask vs --chat: Context Behavior

--ask / "question" --chat
Context across turns No Yes
RAC tracking Single point Full trajectory
Use case One-off questions Conversations

TL;DR: Use --chat when you want BAZINGA to remember previous questions. Each --ask call starts fresh.

AI Commands

bazinga "question" FREE
Ask any question directly. No flags needed.
bazinga "explain quantum entanglement simply"
→ Returns AI-generated answer using best available backend
--ask, -a Alternative to positional
Explicit flag for asking questions. Note: Each --ask is a one-shot call — no context is carried between separate invocations. Use --chat for multi-turn conversations.
bazinga --ask "What is the golden ratio?"
--multi-ai, -m CONSENSUS
Ask multiple AIs and reach consensus through φ-coherence. Uses Groq, Cerebras, Gemini, FreeLLM, Ollama, Claude.
bazinga --multi-ai "Is AI conscious?"
→ 3-6 AIs respond, consensus calculated using Darmiyan Scaling Law (Ψ_D/Ψ_i = φ√n)
--chat RAC
Interactive chat mode with Resonance-Augmented Continuity (RAC). Maintains conversation context across turns, tracks φ-coherence trajectory, and enables true multi-turn dialogue.
bazinga --chat
→ Opens interactive session with memory. Type 'exit' to quit.
--code, -c
Generate code for a given task.
bazinga --code "fibonacci sequence" --lang python
--lang, -l python | javascript | typescript | rust | go
Specify programming language for code generation.
bazinga --code "http server" --lang rust
--quantum, -q
Quantum pattern analysis on text. Extracts φ-coherent patterns.
bazinga --quantum "The golden ratio appears in nature"
--coherence
Calculate φ-coherence score for text.
bazinga --coherence "This text will be analyzed for phi-alignment"
--agent CONSENSUS
AI agent mode that can write code with multi-AI consensus safety.
bazinga --agent "create a todo app"
→ Multiple AIs must agree before code changes are applied
--fresh, -f
Bypass cache and get fresh response.
bazinga --fresh "latest news summary"
--local OFFLINE
Force use of local LLM (Ollama). No internet needed.
bazinga --local "explain recursion"
--file
Provide file context for the question.
bazinga --file main.py --ask "explain this code"

Knowledge Base (KB)

--kb
Search your indexed knowledge base (Gmail, GDrive, Mac files, Phone).
bazinga --kb "meeting notes from last week"
--kb-sources
Show status of all data sources.
bazinga --kb-sources
→ Shows Gmail, GDrive, Mac, Phone indexing status
--kb-gmail
Search Gmail only.
bazinga --kb-gmail "invoice from vendor"
--kb-gdrive
Search Google Drive only.
bazinga --kb-gdrive "project proposal"
--kb-mac
Search indexed Mac files only.
bazinga --kb-mac "research paper"
--kb-phone
Search phone data only. Optionally set path.
bazinga --kb-phone "photos from trip"
→ Or: bazinga --kb-phone ~/path/to/phone/data
--kb-phone-path
Set phone data path and index it.
bazinga --kb-phone-path ~/Android/data
--kb-sync
Re-index all knowledge base sources.
bazinga --kb-sync
→ Re-indexes Gmail, GDrive, Mac, Phone data
--summarize
Use LLM to summarize KB search results into an answer.
bazinga --kb "phi consciousness" --summarize
→ Searches KB, then uses AI to synthesize an answer

Indexing

--index, -i
Index directories for RAG (Retrieval Augmented Generation).
bazinga --index ~/Documents ~/Projects
→ Indexes files, creates embeddings for semantic search
--index-public
Index public knowledge sources (Wikipedia, arXiv, etc.).
bazinga --index-public wikipedia --topics "quantum physics, consciousness"
--topics
Topics for --index-public (comma-separated).
bazinga --index-public wikipedia --topics "AI, machine learning, neural networks"

P2P Network

--phi-pulse NEW v5.7
Start Global Discovery (local Phi-Pulse + HF Registry). Discovers peers on your LAN via UDP broadcast AND registers with the global HF Space registry for cross-internet discovery.
bazinga --phi-pulse
→ Local: UDP broadcast on port 5150
→ Global: Registers at bitsabhi515-bazinga-mesh.hf.space
--join
Join the BAZINGA P2P network with full Kademlia DHT.
bazinga --join
→ Connects to bootstrap nodes, starts DHT, NAT traversal
bazinga --join 192.168.1.100:5151
→ Join via specific peer address
--port default: 5151
Set P2P listening port. Use with --join or --phi-pulse.
bazinga --phi-pulse --port 5152
→ Run on custom port (for multiple instances)
--node-id
Set custom node ID (auto-generated if not provided).
bazinga --phi-pulse --node-id mynode123
--peers
Show connected peers from local and global registry.
bazinga --peers
→ Shows local node info + HF Registry stats + peer list
--nat
Run NAT traversal diagnostics. Detects NAT type, public IP, and hole-punch capability.
bazinga --nat
→ Shows NAT type, public IP, relay eligibility
--sync
Sync knowledge with the network.
bazinga --sync
--publish
Publish your knowledge to the DHT.
bazinga --publish
--query-network
Query the network for knowledge.
bazinga --query-network "distributed consensus algorithms"
--learn
Show federated learning status and configuration.
bazinga --learn
→ Shows adapter rank, version, φ-weighted aggregation info

P2P Discovery Layers

Layer Method Scope Command
1. Local Phi-Pulse UDP (port 5150) Same network/LAN --phi-pulse
2. Global HF Space Registry Any internet --phi-pulse (auto)
3. DHT Kademlia + NAT Decentralized --join

TL;DR: Use --phi-pulse for quick discovery on LAN + global. Use --join for full P2P with Kademlia DHT and NAT traversal.
